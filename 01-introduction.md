## Background

This isn't a Go language tutorial—head to the [official Go Tour](https://tour.golang.org/) or [Effective Go](https://go.dev/doc/effective_go) for that.

## Software Engineering Best Practices

- **Version Control**: Git for tracking changes.
- **Linting**: Catch errors early.
- **Formatting**: Consistent style.
- **Unit Testing**: Verify logic.
- **CI/CD**: Automate builds/tests.

## Tools Overview

We'll use **mise** for Go version/dependency management ([GitHub](https://github.com/mitchellh/mise)).

Install mise:
- macOS: `brew install mise`
- Linux: curl script from docs
- Windows: `winget install mise`

**VS Code** ([GitHub](https://github.com/microsoft/vscode)) with Go extension.

*These are solid modern choices—alternatives like task (Makefile replacement), staticcheck exist.*
